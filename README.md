# 🦷 DENTÉ — лендинг стоматологической клиники

Современный адаптивный лендинг частной стоматологической клиники, ориентированный на доверие, понятную подачу услуг и быструю запись через WhatsApp.

Проект включает восемь смысловых секций, интерактивный калькулятор стоимости, карточки услуг и врачей, отзывы, результаты лечения и форму записи. Весь контент клиники централизован и подготовлен для быстрой замены.

## ✨ Ключевые возможности

- Адаптивный mobile-first интерфейс для смартфонов, планшетов и компьютеров.
- Восемь полноценных секций коммерческого лендинга.
- Основные CTA-кнопки с переходом в WhatsApp и подготовленным сообщением.
- Шесть карточек стоматологических услуг с ориентировочными ценами.
- Интерактивный трёхшаговый квиз-калькулятор стоимости лечения.
- Локальная валидация формы записи без сохранения персональных данных.
- Карточки врачей, отзывы пациентов и примеры работ «до/после».
- Централизованный заменяемый контент в `src/content/clinic.ts`.
- Видимые отметки для неподтверждённых плейсхолдеров.
- Доступная навигация с поддержкой клавиатуры и заметным фокусом.
- Адаптация под WCAG 2.2 AA и автоматические axe-проверки.
- Unit, component и end-to-end тестирование.
- Автоматический аудит обязательного контента перед публикацией.

## 🛠️ Стек технологий

- **React 19** — пользовательский интерфейс.
- **TypeScript** — строгая типизация компонентов и контента.
- **Vite** — локальная разработка и production-сборка.
- **CSS** — собственная адаптивная дизайн-система без UI-фреймворка.
- **Vitest** — unit и component тесты.
- **React Testing Library** — тестирование пользовательских взаимодействий.
- **Playwright** — end-to-end и responsive тесты.
- **axe-core** — автоматизированная проверка доступности.
- **ESLint** — статический анализ исходного кода.

## 🚀 Локальный запуск

### Требования

- [Node.js](https://nodejs.org/) версии **22.17+** или **24 LTS**.
- npm.

### Установка

```bash
git clone https://github.com/Su0pPanda/REPOSITORY-NAME.git
cd REPOSITORY-NAME
npm install
```

> Замените `REPOSITORY-NAME` на название репозитория после его публикации.

### Запуск в режиме разработки

```bash
npm run dev
```

После запуска Vite выведет локальный адрес проекта, обычно `http://localhost:5173`.

### Production-сборка

```bash
npm run build
npm run preview
```

Готовые файлы сборки будут размещены в каталоге `dist/`.

## ✅ Проверка проекта

```bash
# Проверка типов
npm run typecheck

# Статический анализ
npm run lint

# Unit и component тесты
npm run test

# E2E, responsive и accessibility тесты
npm run test:e2e

# Аудит контентных плейсхолдеров
npm run audit:content
```

Для первого запуска E2E-тестов установите Chromium:

```bash
npx playwright install chromium
```

## 📝 Настройка контента

Основные данные клиники находятся в файле:

```text
src/content/clinic.ts
```

Перед публикацией замените:

- название и логотип клиники;
- номер телефона и WhatsApp;
- адрес и режим работы;
- цены и условия предложений;
- сведения и фотографии врачей;
- отзывы и изображения «до/после»;
- ссылки на юридические документы.

Проверить оставшиеся плейсхолдеры можно командой:

```bash
npm run audit:content
```
